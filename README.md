## Cube Tower — простая UI-игра на Unity: горизонтальный скролл блоков внизу, перестановка блоков в правой верхней части в виде башни, выбрасывание в дыру слева сверху. Сохранение состояния башни между сессиями.

## Ключевые моменты
•	Unity: проект открывается как обычный Unity-проект (версия Unity не фиксирована в репо — использовать совместимую с .NET 4.7.1).
•	DI: используется Zenject (контейнер привязок в GameInstaller.cs).
•	Сохранение: состояние сохраняется в PlayerPrefs (ключ берётся из saveKey).
•	Локализация: Localization.cs — поддержка RU/EN, язык задаётся в language.
## Как запустить (быстро)
1.	Открой проект в Unity.
2.	Убедись, что сцена с установщиком (Installer) в проекте загружена и GameInstaller привязан в сцене.
3.	В инспекторе: у GameInstaller назначь ссылки на configService, saveService, bottomScrollView, towerView, holeView, messageView.
4.	Проверь GameConfig (в GameConfig.cs) — размеры блоков, количество в скролле, параметры анимаций, language.
5.	Запусти сцену в редакторе.
## Управление и поведение
•	Перетаскивание: тащить блок из нижнего скролла (или из самой башни). Drag/Drop реализован в BlockView.
•	Поставить в башню: отпустить над towerArea (правый верх). Первый блок можно ставить, перед этим towerRoot перемещается в точку дропа.
•	Удаление блока: бросить блок в holeView (левая область) — для блоков с тегом TowerBlock удаляется именно блок из башни, для ScrollBlock выполняется анимация промаха и блок возвращается в скролл.
•	Сообщения: messageView показывает текст (локаль из Localization).
## Важные замечания / настройки
•	Теги: добавь в Tag Manager теги TowerBlock и ScrollBlock. Префаб нижнего блока должен иметь ScrollBlock; блоки в башне помечаются как TowerBlock.
•	Canvas: при проблемах с позиционированием проверь Canvas.renderMode (код учитывает Overlay vs ScreenSpaceCamera).
•	Сохранение: состояние хранится через JSON в PlayerPrefs (saveService).
•	Если башня уходит за экран, логика пытается ограничить towerRoot и проверяет видимость верхнего блока перед добавлением.
## Структура (главные файлы)
•	GameController.cs — основная логика и обработка драг-н-дроп.
•	BlockView.cs — визуал и поведение блока.
•	TowerView.cs — отображение и управление башней.
•	BottomScrollView.cs — нижний скролл блоков.
•	HoleView.cs — зона дыры.
•	GameConfig.cs — настраиваемые параметры.
•	Localization.cs — тексты сообщений.
•	SaveService.cs — сохранение/загрузка.
